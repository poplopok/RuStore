## О проекте

Приложение реализует витрину приложений с категориями, поиском, карточкой приложения и полноэкранным просмотром скриншотов. Данные загружаются с backend, размещенного на хостинге. Для приложений с доступным APK реализована установка через `PackageInstaller` и удаление через системный uninstall-flow.

## Стек технологий

- Kotlin
- Jetpack Compose (UI)
- ViewModel + StateFlow
- Navigation Compose
- Retrofit + OkHttp + kotlinx.serialization
- Foreground Service (загрузка APK в фоне)
- PackageInstaller API (установка)
- DataStore (флаг первого запуска / онбординг)

## Реализованные экраны и функции

### 1) Онбординг

- Показывается только при первом запуске
- Логотип, приветственный текст, кнопка перехода в витрину

### 2) Витрина приложений

- Список приложений: иконка, название, краткое описание, категория
- Категории: `Финансы`, `Инструменты`, `Игры`, `Государственные`, `Транспорт`
- Клик по карточке открывает экран приложения
- Есть переход к списку категорий
- Поддержка Pull-to-Refresh
- Пагинация кнопкой «Показать еще»

### 3) Экран категорий

- Показ названия категории и количества приложений
- Выбор категории возвращает на витрину с фильтрацией
- Поддержка Pull-to-Refresh

### 4) Экран поиска

- Поисковая строка с автофокусом
- Интерактивный поиск по названию
- При пустой выдаче показываются популярные приложения
- Поддержка Pull-to-Refresh

### 5) Карточка приложения

- Иконка, название, разработчик, категория, возрастной рейтинг
- Горизонтальный список скриншотов
- Полное описание
- Кнопки: установить/открыть/удалить (в зависимости от состояния)
- Поддержка Pull-to-Refresh

### 6) Полноэкранный просмотр скриншотов

- Открытие с выбранного скриншота
- Горизонтальная прокрутка всех скриншотов
- Кнопка возврата назад

### 7) Обработка ошибок

- Для сетевых и других ошибок используются error-state экраны
- Приложение не падает при типовых ошибках загрузки

## Опциональная часть

### Собственный backend

Реализован отдельный backend в формате статического API:

- `GET /backend/apps.json`
- `GET /backend/apps/{id}.json`
- `GET /backend/apks/{id}.apk`
- `GET /backend/media/...`

### Фоновая загрузка APK + установка/удаление

- Загрузка APK через Foreground Service
- Установка через `PackageInstaller`
- Удаление через системный uninstall-flow
- Отображение прогресса загрузки и состояний установки

## Архитектура и структура проекта

```text
app/
  src/main/java/com/example/rustorecatalog/
    ui/                # экраны и UI-компоненты Compose
    viewmodel/         # состояние экранов
    network/           # Retrofit API + конфигурация клиента
    data/              # репозиторий
    install/           # загрузка APK, установка, удаление
    navigation/        # маршруты и NavHost
backend/
  catalog-data.json    # исходный каталог
  apks/                # APK-файлы
  media/               # иконки/скриншоты
  build-static-site.mjs
  static-site/         # результат генерации для хостинга
```
